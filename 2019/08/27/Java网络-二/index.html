然后在运行Client端，Client运行完后回到Server端查看运行结果：

说明访问成功。这里可能会遇到一个拒绝连接问题，其一是防火墙拦截，其二是端口没有被打开。

但这只能使一个客户端进行一次连接,服务器端不能只接收一个一个客户端的连接后就停止了,例如微博的服务器端,不可能是你访问了其他人就没法在访问了,服务端就停了,所以服务器端应该一直运行,要想让它一直在运行,就给它套一个死循环,让其一直接收客户端的连接。
